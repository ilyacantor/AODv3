3) Run a “two-path diff” test: reconcile code vs pipeline code

Most of these bugs are not “domain extraction is wrong”—they’re “a different function path is used depending on mode.”

So test this explicitly:

Path A: the function used by the reconciliation report generator

Path B: the function used by the AOD pipeline when it builds asset keys

For the same list of raw domains, print:

raw_domain | canonical_key(reconcile) | canonical_key(pipeline) | match?

If any row differs, that’s the root cause class:

“two canonicalizers exist”

“imports differ”

“mode flag chooses a different canonicalizer”

“fallback path returns raw domain”

This is exactly consistent with your earlier suspicion about _extract_registered_domain() returning the raw string. (And the JSON symptoms match that failure mode.)
