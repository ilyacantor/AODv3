The Single Root Cause: CMDB/IdP Correlation Contract Mismatch
What Farm does:

Generates CMDB entries with canonical_domain field linking to discovery domain
Generates IdP entries with canonical_domain field linking to discovery domain
Expected block uses canonical_domain for correlation → marks asset as HAS_CMDB/HAS_IDP → GOVERNED
What AOD apparently does:

Uses different correlation logic (name matching? external_ref parsing? unknown)
Does NOT find the same CMDB/IdP correlations
Marks asset as NO_CMDB/NO_IDP → SHADOW_CLASSIFICATION
