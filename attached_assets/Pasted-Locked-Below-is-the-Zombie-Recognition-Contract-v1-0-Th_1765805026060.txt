# AOD Zombie Recognition Contract — v1.0 (Deterministic)

## Scope

Phase 0 only.
Purpose: **fact-based identification of Zombie assets**.
No ML. No scores. No inference. No curves.

---

## Inputs (from Farm → AOD)

### Required

* `run_id` (string) — grading key
* `snapshots[]` — one or more snapshots, all with:

  * `observed_at` (timestamp, from upstream evidence)
  * `asset_id`
  * `asset_type`
  * `source_system`
  * `evidence[]` (may be empty, but must exist)

### Configuration

* **Zombie window**: `X days` (fixed, explicit, passed to AOD at runtime)

  * Example: `X = 90`
  * No defaults. Must be provided.

---

## Definitions (precise)

### Asset Exists

An asset **exists** if it appears in **any** system of record within the snapshot:

* CMDB
* Billing
* IdP
* Cloud inventory
* Secrets / integration registry

> Existence ≠ activity.

---

### Activity Evidence

Activity evidence is **any event** that proves use or interaction, with a valid `observed_at`, from:

* IdP login / token use
* Access logs / API calls
* Billing usage / metered events
* Job execution / workflow runs

Configuration records alone (CMDB entries, config state) **do not count** as activity.

---

### Zombie Asset (binary)

An asset is a **Zombie** if and only if **all** conditions are true:

1. **Exists**
   Asset appears in ≥1 system of record.

2. **No activity evidence within window**
   No `observed_at` for that asset **≥ (now − X days)** across all evidence sources.

3. **Evidence exhaustiveness**
   All configured evidence sources were checked (must be enumerated).

If any condition is false → **NOT a Zombie**.

---

## Explicit Non-Rules (forbidden)

* ❌ Anomaly scores
* ❌ Percentile cutoffs
* ❌ “Low usage” heuristics
* ❌ Inferred last_seen
* ❌ generated_at used as activity
* ❌ Confidence / probability outputs

---

## Output (AOD → Farm)

For each asset evaluated:

```json
{
  "run_id": "...",
  "asset_id": "...",
  "classification": "ZOMBIE" | "NOT_ZOMBIE",
  "window_days": X,
  "evidence_checked": [
    "IdP",
    "AccessLogs",
    "Billing",
    "CMDB"
  ],
  "last_activity_observed_at": "timestamp | null",
  "reason": "Plain-English, one sentence, factual"
}
```

### Reason examples

* “Asset exists in CMDB and Billing; no activity observed in IdP, logs, or billing within 90 days.”
* “Recent IdP token usage observed on 2025-11-02.”

---

## Grading Rules (Farm)

Farm grades **per run_id**:

* PASS if AOD:

  * Classifies every asset deterministically
  * Uses only evidence provided
  * Correctly applies the X-day window
* FAIL if:

  * Any zombie is misclassified
  * Any activity is inferred
  * Any required field is missing
  * Output references non-existent evidence

---

## Stability Guarantee

This contract **does not change** until:

* Mono-dynamic snapshots are enabled, or
* Shadow IT contract is introduced

No silent extensions.
