### Step 2 — Implement Primary Key Freeze (single choke point)

Locate the single place where the emitted asset key is chosen (the “primary key”).

Implement function:

`choose_primary_key_from_observations(observations) -> registrable_domain`

Rules:

1. Extract candidate domains from **observations only** (hostname/domain fields).
2. Normalize to registrable domain (eTLD+1).
3. Score candidates:

   * higher `support_count` wins
   * tie-breaker: higher `source_diversity`
   * tie-breaker: most recent activity
4. Output `primary_key`.

Then enforce immutability:

* set `asset.primary_key` exactly once
* disallow any later step from changing it (merge/alias must not overwrite)

**Hard guardrail invariant (must fail fast):**
If any code path attempts to emit an asset under a key different from `asset.primary_key`, raise and fail the run.

Allowed behavior:

* `identifiers.domains` can include additional domains
