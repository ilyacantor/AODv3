How AOD Works

What is AOD?

AOD (AutonomOS Discover) finds and catalogs the software systems your organization uses. It analyzes evidence from multiple sources - login records, cloud inventories, network logs, and more - to build a complete picture of your software landscape.

The 6 Numbers Explained

Observations
Raw sightings of software from various sources. Think of these as "we saw this software somewhere" - not yet verified.
Assets
Confirmed software systems that passed quality checks. These are the real applications your organization uses.
Artifacts
Non-system items found during discovery - dashboards, reports, spreadsheets. Useful to track, but not software systems.
Findings
Issues or gaps discovered - like "this app has no owner" or "not found in your CMDB." Things that need attention.
Ambiguous
Cases where we found multiple possible matches and couldn't decide. These need a human to review and pick the right one.
Rejected
Items we saw but excluded from the catalog - didn't meet the quality bar or weren't real software systems.
Shadow
Unmanaged assets - software in use that isn't tracked in official systems like CMDB or Finance.
Zombie
Inactive assets - software still in records but showing no recent usage or activity.

Using Drill-Down

  • Click any number to explore the data behind it
  • Filter by category - click groups to narrow down (e.g., by type, vendor, or source)
  • Keep drilling until you reach individual items
  • Navigate back using the breadcrumb trail or Back button

Getting Started

  • Select a tenant from the dropdown (your organization)
  • Pick a snapshot - a point-in-time collection of evidence
  • Click "Fetch & Run Discovery" to analyze the data
  • Review the results - explore assets, findings, and drill into the numbers
